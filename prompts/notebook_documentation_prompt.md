# Prompt para documentar un notebook

```text
Quiero mejorar la documentación Markdown de este notebook de Google Colab.

Contexto del notebook:
- Objetivo:
- Dataset:
- Técnicas usadas:
- Resultado principal:
- Limitaciones:

Fragmento de código o sección a documentar:
```python
[pega aquí el bloque de código]
```

Ayúdame a escribir una explicación clara para una celda Markdown que incluya:
1. qué hace esta sección;
2. por qué es necesaria;
3. qué decisiones técnicas se toman;
4. qué debe mirar una persona al revisar el resultado;
5. posibles limitaciones.

Tono:
- claro;
- profesional;
- sin exagerar;
- orientado a aprendizaje;
- útil para alguien que está revisando mi portfolio.
```

## Uso recomendado

Este prompt ayuda a convertir notebooks que parecen “solo código” en notebooks que muestran razonamiento técnico.
